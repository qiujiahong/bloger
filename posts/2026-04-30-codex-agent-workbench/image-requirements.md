# Codex Agent 工作台配图需求

## 整体要求

- 统一视觉风格：清新手绘风，线条干净，颜色明亮但不刺眼，适合微信技术号、掘金、华为开发者论坛正文阅读。
- 视觉关键词：工程工作台、代码仓库、CLI、IDE、PR review、测试、团队协作、权限边界、审计记录。
- 色彩建议：浅灰白背景，辅以薄荷绿、科技蓝、珊瑚橙、石墨黑。避免大面积深色、紫蓝渐变、赛博朋克风和营销海报感。
- 字体建议：中文使用清晰黑体或手写感无衬线，标题字数少，正文图中的标注保持可读。
- 所有图片要和 `blog.md` 中的 Markdown 占位符一一对应，默认输出到 `./assets/`。
- 不要出现未经授权的 OpenAI 官方 logo 变体，不要画成“机器人替代程序员”，不要暗示已经完成无人化交付。

## 内部架构图：`architecture-codex-agent-workbench.png`

- 标题：`Codex Agent 工作台：从本地入口到团队工程闭环`
- 目标：读者一眼看懂 Codex 不只是 CLI，而是在多个工程触点之间形成工作台。
- 节点层级：
  - 入口层：`CLI`、`IDE Extension`
  - 上下文层：`本地代码库`、`文件变更`、`命令输出`、`测试结果`
  - 执行层：`代码修改`、`运行测试`、`解释错误`、`生成文档`
  - 协作层：`PR 评论`、`安全审查`、`团队策略`
  - 护栏层：`权限边界`、`命令白名单`、`审计记录`、`人工确认`
- 关系与流向：
  - 入口层读取上下文层。
  - 执行层基于上下文产生 diff、测试反馈和说明。
  - 协作层接收执行层结果，并进入 PR 或团队 review。
  - 护栏层横向覆盖所有层，表示不是事后补丁，而是全流程规则。
- 视觉建议：分层架构图，横向或纵向均可；每层用不同浅色底带，节点用手绘小卡片，但不要做成厚重卡片堆叠。
- 标注：在图底部加一行小字：`方向清楚，成熟度仍需以实际试点验证。`
- 可截图收藏：是。信息密度要高，但字不要小到手机端无法阅读。

## 逻辑示意图：`logic-cli-to-workbench.png`

- 标题：`判断 Codex 变化的三条信号`
- 要表达的核心逻辑：
  - `官方产品线`：CLI 与 IDE extension 同属 Codex 产品线。
  - `团队叙事`：Codex for teams 把 GPT-5.5、Code Red、安全审查、PR 评论放进团队工程语境。
  - `快速迭代`：2026-04-29 GitHub releases 连续发布多个 Rust alpha 版本。
  - 三条信号汇聚到结论：`从个人 CLI 走向团队 Agent 工作台`。
- 读者看完应理解：
  - 这不是单个 release 的新闻搬运，而是产品形态变化。
  - 证据支撑的是方向判断，不是对所有能力成熟度的背书。
- 视觉建议：三条支流汇入一条主河流，或三张便签汇入中心结论。保持手绘风，不做复杂商业咨询图。
- 文案限制：每个节点不超过 18 个汉字，避免堆满原文。

## 高信息密度正文图：`checklist-codex-team-adoption.png`

- 标题：`Codex 进入团队流程前，先过这张检查表`
- 标注：`可截图收藏`
- 形式：四象限或分栏 checklist，适合手机截图保存。
- 内容结构：
  - `适合先试`
    - 需求澄清
    - 代码定位
    - 小范围修改
    - 测试失败分析
    - PR 前自检
    - 文档更新
  - `暂时谨慎`
    - 大面积重构
    - 生产配置改动
    - 密钥或权限相关代码
    - 无测试覆盖的核心链路
    - 自动合并 PR
  - `必须设置`
    - 仓库访问边界
    - 命令白名单
    - 人工确认点
    - 审计记录
    - 回滚方案
  - `评估指标`
    - 节省时间
    - 返工次数
    - review 命中问题
    - 测试通过率
    - 权限收紧项
- 视觉建议：清新手绘风的行动指南图，信息密度高但层次清楚；每栏用不同浅色标题条，配小图标。
- 尺寸建议：`1400 x 1800`，方便小红书与微信正文截图阅读。

## 平台适配

- 微信技术号：
  - 封面优先使用横版 `1200 x 675`。
  - 正文图宽度建议不小于 `1200px`。
  - `checklist-codex-team-adoption.png` 可作为文末收藏图。
- 小红书：
  - 封面裁切成竖版 `1242 x 1660`。
  - 标题文字控制在两行内，突出 `Agent 工作台` 和 `工程闭环`。
  - checklist 图可拆成两张竖图，但内容顺序保持一致。
- 华为开发者论坛：
  - 封面使用横版 `1200 x 630`。
  - 架构图与逻辑图优先保证术语清晰，不追求强装饰。
